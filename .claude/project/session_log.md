## 2026-07-18 · Full-stack (main) — Live end-to-end audit + 2 real bug fixes
Ran the actual app (uvicorn on 8000 + vite preview on 4173) and drove the FULL recruiter journey over real HTTP (`scratchpad/e2e_journey.py`, 46 checks): upload JD+3 resumes → evaluate → report (authenticity, profile, must/nice, evidence) → history → PDF (10.9KB real %PDF) → batch rank → all 8 analytics sections → 3 dashboard endpoints → workflow GET/PATCH/persist/history → notes CRUD. Found & fixed **two real bugs the test suite missed**: (1) **Vector index contamination** — `has_existing_index(resume_id)` reused a stale on-disk FAISS index whenever a resume_id collided across DBs, so evaluations ran against the WRONG resume. Fixed by content-addressing: `compute_fingerprint` (sha256) + `has_valid_index` + sidecar; agent rebuilds on mismatch (`vector_store_service`, `agents/candidate_intelligence`). (2) **Evidence-quality / floor too low** — `RETRIEVAL_MIN_SIMILARITY=0.30` let BGE-small's loose cross-domain similarity (~0.56-0.61) pass, so a React frontend resume spuriously matched every backend requirement (74/100, 0 missing). Measured per-req cosine separation across 3 resumes; raised default to **0.62** (genuine matches ~0.62+, spurious ~0.56-0.61). After fix: strong=65 / medium=44 / weak=**0**, frontend resume correctly 0/8 matched. Verify: backend **pytest 136 pass** (+3 fingerprint), frontend **build+lint clean**, live **E2E 46/46**, frontend serves SPA+bundle (200). No browser-automation tool available → no pixel screenshots (verified at HTTP/contract/build/serve layer). Not committed.
